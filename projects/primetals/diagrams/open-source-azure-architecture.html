Primetals PoC Architektur mit minimalem Lock-in

Offene Frameworks für Ingestion, Indexierung und Retrieval; Azure als Cloud- und Security-Fundament

Audience: Kunde, Architekt, Engineering Level: Container + austauschbare Komponenten Goal: Portabilität, offene Datenformate, klare Adapter

RAG APIs, Recommender, UI

Azure bleibt Fundament

  • Cloud SubstrateAKS oder Container Apps, Blob / ADLS, ACR, VNet, Private Link
  • Managed DataAzure Database for PostgreSQL für pgvector und Metadaten
  • SecurityKey Vault, Managed Identity, Network Isolation

Austauschbare Komponenten

  • OCRRouter kapselt Docling, Tesseract, PaddleOCR oder Azure DI
  • ModelsLiteLLM / OpenAI-compatible APIs kapseln Providerwechsel
  • IndexRepository Interface für pgvector, Qdrant, OpenSearch oder Azure AI Search

Portabilitäts-Regeln

  • FormatsParquet, JSON, Markdown und OCI Images statt proprietärer Payloads
  • ContractsFastAPI/OpenAPI, typed schemas, explicit adapters
  • Ops Trade-offWeniger Service-Lock-in, dafür mehr Betrieb und Modellverantwortung
Compute RuntimeAKS oder Container Apps für containerisierte Services und Workers
Container SupplyAzure Container Registry, OCI Images, GitOps optional
SecretsKey Vault plus External Secrets oder workload identity
Network BoundaryVNet, Private Link, private endpoints, project isolation
OT
ObservabilityOpenTelemetry, Prometheus, Grafana, Loki; Azure Monitor optional

Minimal-Lock-in Prinzip: Azure liefert Betrieb, Security und Managed Infrastructure; fachliche Intelligenz, Datenformate, Retrieval-Verträge und Modellzugriffe bleiben über offene Frameworks und Adapter austauschbar.
